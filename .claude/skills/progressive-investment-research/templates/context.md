# {{TITLE}} Context

## 工作区定位

这是一个累进性研究 dossier。用 1-3 句话说明研究对象、边界和当前用途。

## 默认 Skill / Workflow

- 默认使用：`progressive-investment-research`
- 研究结论事实源：`current-synthesis.md`、`model-map.md`、`open-questions.md`、`modules/`、按需 `models/`
- `context.md` 只负责接续入口和工作约定，不替代 Current Model。

## 接续入口

每次接续时优先读取：

1. `context.md`
2. `current-synthesis.md`
3. `model-map.md`
4. `open-questions.md`
5. 与当前问题相关的 `modules/` 或 `models/`

## 写入规则

默认先讨论、澄清和回答，不把每次对话都写入 dossier。

只有以下情况才更新 `context.md`：

- 新建 dossier 或恢复成本过高。
- 用户明确要求写入或更新 context。
- 研究边界、默认 skill、入口文件、路径、工具或验证方式变化。
- 某条工作约定会影响后续所有接续。

研究结论、证据、数字、假设和开放问题应写入对应模型文件，不写入这里。

## 本地约定

记录必要的本地路径、验证命令、权限限制或用户偏好。

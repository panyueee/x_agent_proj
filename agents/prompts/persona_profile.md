# 人物画像提示词（map-reduce 两段，由 profile.py 按二级标题切分）

## MAP

你是一名研究"卖方首席分析师思维方式"的分析员。下面是分析师【{person}】在一段时间内发表的若干篇文章（每篇已截断）。请通读后输出**批级笔记**，供后续汇总成完整画像。

对每篇文章提取（没有就写"无"）：
1. 核心观点/结论（一句话）与方向（看多/看空/中性/结构性分化等）；
2. 论证逻辑链（他/她是从什么数据、什么框架推出结论的）；
3. 显式预测（对未来的可验证判断，注明文章日期）；
4. 方法论线索（惯用的数据源、分析框架、拆分维度）；
5. 相对此前立场的变化迹象（如有）。

输出格式：Markdown 列表，按文章日期排列，每篇以「日期 | 标题」开头。保持精炼，总量不超过 1500 字。

## REDUCE

你是一名研究"卖方首席分析师思维方式"的资深分析员。下面是对分析师【{person}】截止 {until_date} 全部语料的批级笔记。请汇总成一份结构化画像，摸透其逻辑发展与底层思维。

严格输出一个 JSON（用 ```json 代码块包裹），字段如下：

```json
{{
  "person": "姓名",
  "until_date": "画像截止日期",
  "analytical_framework": ["分析框架与方法论要点，每条一句话，5-10 条"],
  "worldview": ["核心世界观假设（其判断背后不言自明的前提），3-8 条"],
  "timeline": [
    {{"period": "时间段", "stance": "重大立场", "shift": "相对之前的转变（无则空字符串）"}}
  ],
  "explicit_predictions": [
    {{"date": "文章日期", "prediction": "预测内容", "verifiable_by": "何时/如何可验证"}}
  ],
  "style": ["写作与推理模式（结构习惯、数据偏好、语言风格），3-6 条"],
  "summary": "200 字以内的画像总结：这个人如何思考、当前最关注什么、下一步最可能写什么"
}}
```

要求：
- 一切内容必须有笔记依据，不要脑补；
- timeline 按时间排列，突出"转变点"；
- explicit_predictions 只收录可后验的判断。

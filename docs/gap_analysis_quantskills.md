# QuantSkills 三个开源 skill 查缺补漏分析

> 分析日期：2026-07-03　范围：只读分析，不改代码、不 commit。
> 对象：`skill-market-daily-review`（每日复盘）、`skill-futures-deepview-analyst`（期货深度）、`skill-a-share-stock-dossier`（A股个股档案）。三者均 QuantSkills 出品、GPL/MIT 系开源、markdown 为主，可自由借鉴。
> 结论先行：**三个 skill 全是 Pandadata 数据接口的薄封装**（每个 frontmatter 都 `requires: skill-pandadata-api`，scripts 只是 Pandadata 调用管道）。本项目有自己的数据层（akshare / 东财 F10 / 天眼查 / 问财 / TradingView），所以对我们几乎没有"装 skill"的价值——**要抄的是它们的分析框架、输出结构、风险规则，不是装它们**。全篇按"**数据层缺口** vs **框架层缺口**"两条线拆开评估，避免把 akshare 就能喂的数据说成缺口，也避免把"我们有数据"当成"我们有方法"。

## 一、逐 skill 精读

### 1. skill-market-daily-review（A股每日收盘复盘）

**做什么**：收盘后一句话生成 A 股当日复盘报告，每个数字可溯源到接口 + 数据日，缺数据宁可标注不编造。

**核心工作流**（SKILL.md）：
1. 定交易日：`get_last_trade_date` / `get_trade_cal`，休市则只回"今日休市"。
2. 按固定顺序取数：交易日历与股票池 → 指数行情与估值 → 市场宽度与情绪 → 行业概念热点 → 龙虎榜/大宗/两融/北向持股。
3. 从原始行**自己算**衍生指标：涨跌家数、涨停跌停家数、成交额龙头、行业/概念领涨、龙虎榜净买卖前列、大宗折溢价分布、两融余额变化、北向持股增减。
4. 套 `references/report-template.md` 生成 Markdown，存 `reports/daily/YYYYMMDD.md`。
5. **跑 `scripts/validate_report.py` 校验**，缺章节/缺来源/缺数据日必须补齐才交付。

**值得注意的 references / 方法细节**：
- `report-template.md`：一份完整的**盘后固定输出结构**——摘要（指数/宽度/热点/资金四句）→ 指数概览与估值（含估值分位）→ 市场宽度与情绪 → 行业与概念热点 → 龙虎榜与大宗 → 两融与北向 → 异动与风险 → 数据说明（用了哪些接口、数据截止、缺失降级、统计口径）。
- `pandadata-map.md`：给出**降级策略**——全市场/概念聚合太慢或不可用时，保留交易日/指数/估值/龙虎榜，市场宽度直接标"不可用"而**不估算**，跳过的接口写进"数据说明"。
- 规则硬约束：涨停跌停必须写清口径（是否含 ST、是否含一字板）；两融/北向标 T+1；不给明日操作建议。

**scripts**：`validate_report.py`——纯正则校验器，检查 9 个必需章节（标题含"复盘/收盘"、摘要、指数、市场宽度、行业/概念、龙虎榜/大宗、两融/北向、异动/风险、数据说明）、数据来源说明、数据日说明、两融北向的 T+1 标注、非投资建议声明；报告 <500 字判为不完整。**这是三个 skill 里唯一真正可复用、与 Pandadata 无关的脚本。**

**agents**：`portable-loader.md`（给不原生认 SKILL.md 的 agent 用的加载提示词）+ `openai.yaml` / `cursor-rule.mdc`（各平台入口壳）。无实质子 agent 分工，就是同一套逻辑的多平台适配。

### 2. skill-futures-deepview-analyst（期货深度分析）

**做什么**：把"分析螺纹钢席位博弈""看豆粕期限结构和仓单""扫哪些品种有跨期套利"这类自然语言，转成 Pandadata 期货 DeepView 调用计划，输出**事实与推断分离**的中文研判。

**核心方法 / 接口地图**（SKILL.md 的 Interface Map + playbook）。它把期货分析切成 6 个模块群，这套**分析框架**是本 skill 最大价值：
- **席位博弈**：净持仓保证金（多空差）、净持仓变化、建仓过程、席位评级、盈亏排行、净持仓排名 → 找价格/持仓/资金的**背离**（价涨但净多减弱 = 背离），看主力集中度。
- **盈亏与资金流**：盈利席位是加仓还是减仓、亏损席位是否被迫平仓、大额资金流是否与净持仓方向一致、信号是单一席位主导还是普遍。
- **基差与期限结构**：基差 + 基差率、基差在回看窗口的**分位**、曲线形态（contango/backwardation/局部倒挂/陡峭化）、同一交易日的跨期价差变化。
- **库存现货**：仓单/库存的日环比周环比同比、虚实盘比（写清分母口径）、现货利润（生产利润扩张/压缩 → 供给增减）、现货报价是否支撑期货升贴水叙事。
- **全品种扫描**：先用排行/池子接口筛选 → 明示过滤条件（流动性/排名阈值/同日可得）→ 取 top 5-10 → 只对候选做二级验证。

**值得注意的方法细节**（`analysis-playbook.md`）：
- **同一交易日对齐**：跨合约/跨模块比较必须同 `trade_date`，不同交易日拼成一个信号必须显式标"异步证据"。
- **措辞纪律**："净多增加"是事实，"多头更主动"是推断，"必然上涨"禁止。
- **矛盾处理**：模块冲突时不强凑单一结论，分清证据是价格/持仓/现货/套利哪一类，冲突时降低置信度，并指出下一个能解决冲突的数据点。
- **空数据处理**：先查是否交易日、日期字段类型（trade_date vs start/end）、品种/合约/交易所后缀格式，再套 `get_future_dominant` 已知主力合约兜底，最后如实报"该接口在所选条件下无返回"。

**scripts**：无独立脚本（引用了 `scripts/search_api_docs.py`，属 pandadata-api skill）。
**agents**：同 daily-review，portable-loader + 平台壳，无实质子 agent 分工。

### 3. skill-a-share-stock-dossier（A股个股档案）

**做什么**：输入一个 A 股代码，输出可溯源的中文尽调报告：基本面、分红资本运作、股东行为、质押解禁减持风险、资金面。

**核心工作流 / API 阶段图**（`dossier-guide.md`）。5 个阶段：
- **公司概况**：`get_stock_detail` / `get_stock_industry` / `get_share_float` / `get_stock_status_change`（含 ST/特别处理历史）。
- **财务报表**：`get_fina_reports` / `get_fina_performance` / `get_fina_forecast` / `get_audit_opinion`（业绩预告变化、非标审计意见）。
- **分红资本运作**：分红/回购/定增/配股/拆股/投资者活动。
- **股东与事件风险**：十大股东、股东户数、股东变动、质押、质押统计、限售解禁。
- **资金面**：日行情、龙虎榜、大宗、两融、北向持股。

**值得注意的方法细节**：
- **要自己派生的指标清单**：营收/净利/营业利润 YoY、毛利率/净利率/ROE/ROA、资产负债率、经营现金流/净利润匹配度（标负值或持续弱匹配）、分红频率/派息率/隐含股息率、大股东集中度/户数环比、解禁规模占流通盘或市值比、质押率、拟减持规模、多重负面信号是否叠加。派生值必须写明公式与字段名。
- **⭐ 风险规则阈值表（本 skill 最可抄的一块）**：
  - 高：ST/*ST/退市风险；非标审计意见；质押率 ≥50% 或控股股东质押高且上升；未来 90 天解禁 >流通盘 10%（或 >市值 5%）；大股东减持计划叠加业绩预告下修/首亏/续亏。
  - 中：质押率 30–50% 或新增大额质押；90 天解禁 5–10% 流通盘；户数增 >20% 而股价平/弱；连续三年不分红却频繁再融资。
  - 低：孤立小事件/日期陈旧/缺分母 → 放附录不进执行摘要。
  - 组合信号要显式描述（如 `质押率高 + 解禁临近`、`减持计划 + 业绩预告下修`）。
- **输出规范**：附录必须有来源表（数据模块/来源接口/查询窗口/返回行数/最新日期报告期/备注）；空数据保留标题写"无数据 + 接口 + 窗口"；固定免责声明。

**scripts / agents**：引用 `scripts/call_api.py`（pandadata-api 管道）；agents 同上，无实质子 agent 分工。

## 三、真缺口清单（确实没有、且值得补）

按"数据 or 框架"标注。**注意：这些 skill 依赖 Pandadata，我们不会装它们；补法一律是"用 akshare/东财喂数据 + 抄它的方法/结构进我们代码"。**

1. **【框架，最大】期货分析层**：我们只有期货历史价格 parquet，完全没有分析框架。skill 的 6 模块切法（席位博弈/盈亏资金流/基差期限结构/仓单库存现货/全品种扫描）+ 背离判定 + 同日对齐纪律非常成熟，直接可抄。数据侧 akshare 提供持仓排名、基差、库存等大部分输入——**这是框架缺口不是数据缺口**。

2. **【框架+数据】结构化盘后复盘**：digest 是信号聚合，缺"每个交易日固定动作 → 固定结构 → 可溯源"的收盘复盘。框架抄 `report-template.md`；数据侧要给 finance_fetcher 补：市场宽度（涨跌/涨停跌停家数）、两融余额、大宗交易、指数估值分位——全部 akshare 可得。

3. **【框架，高性价比】报告输出 QA 校验器**：三个 skill 里唯一与数据源无关、可直接复用的脚本 `validate_report.py`。本项目 digest/dossier **无任何输出质量门禁**。它强制的纪律（每个数字可溯源到接口+日期、章节完整、免责声明、缺数据显式标注）正是我们缺的。看似"就是个正则脚本"，价值在**它固化的产出纪律**。

4. **【框架 ⭐】个股事件风险规则表**：dossier-guide 的高/中/低阈值表（质押率≥50%、90天解禁占流通盘比、减持×业绩下修叠加、户数增+股价弱、连续不分红却频融资）。我们的 risk 模块是组合层 Aladdin 风险分解，**没有个股事件风险规则打分**。规则本身可直接抄进 entity_resolver/dossier，数据侧需补质押/解禁/减持事件流（akshare 可得）。

5. **【框架】溯源与事实-推断分离纪律**：三个 skill 共同的方法论——事实/派生/判断三分层、派生值写公式字段名、附录来源表、推断用"可能提示/需关注"而非断言、措辞禁"必然"。抄进 digest/dossier 的产出规范，几乎零成本，直接提升可信度与可审计性。

> 诚实边界：个股档案的**基础设施**（Obsidian promote 审计、证据行、entity_resolver 跨源主键、persona 人物建模）我们比该 skill 强，不算缺口；龙虎榜、北向净流入、十大股东我们已有；backtest/factor_model/psych/policy_events 更是它们完全没有的。缺的只是上面这些**框架与个别数据字段**。

## 四、借鉴优先级（性价比排序 + 接入点）

| 优先级 | 借鉴项 | 类型 | 性价比 | 接入点 |
|---|---|---|---|---|
| **P0** | 报告 QA 校验器 | 抄方法 | 极高（半天） | 新增 `scripts/validate_report.py` 泛化版，门禁 digest.py / dossier 产出；每个数字必带来源+日期、免责声明、章节完整 |
| **P0** | 溯源 + 事实/推断分离纪律 | 抄方法 | 极高（改产出模板） | digest.py 各 section 加"数据说明/来源接口/数据日"；dossier 加附录来源表 |
| **P1** | 结构化盘后复盘 | 抄结构+补数据 | 高 | `report-template.md` 结构接进 digest.py 新增"A股收盘复盘"section；finance_fetcher 补 breadth/两融余额/大宗/估值分位（akshare） |
| **P1** | 个股事件风险规则表 | 抄方法 | 高 | 阈值表接进 entity_resolver/dossier 的个股风险信号；补质押/解禁/减持数据流 |
| **P2** | 期货分析框架 | 抄方法+补数据 | 中（体量大） | 新增 `x_agent/futures_analyst.py`，6 模块切法 + 背离判定 + 同日对齐；数据用 akshare 持仓/基差/库存 |

**一律"抄方法进我们代码"，不"装 skill"**——因为装它们=接管 Pandadata 数据层，与本项目自有数据层冲突，收益为负。

## 五、一句话小结

- **每日复盘 skill**：一套可溯源、有降级策略、带输出校验器的 A股盘后固定复盘流水线——我们缺的是"固定结构 + QA 门禁 + 几个数据字段"，不是数据源。
- **期货深度 skill**：把期货研判切成席位/基差期限结构/仓单库存/全品种扫描的成熟分析框架——我们期货只有价格，这是纯框架缺口。
- **A股个股档案 skill**：规则化个股尽调（财务派生 + 事件风险阈值表 + 溯源附录）——档案基础设施我们更强，缺的是"事件风险规则表 + 财务/质押解禁减持数据"。

**Top 5 真缺口**：①期货分析框架 ②结构化盘后复盘 ③报告 QA 校验器 ④个股事件风险规则表 ⑤溯源/事实-推断分离纪律。

**最该先做的 3 件（高性价比）**：
1. 抄 `validate_report.py` → 泛化成 digest/dossier 的输出 QA 门禁（P0，半天）。
2. 抄溯源 + 事实/推断分离纪律 → 进 digest/dossier 产出模板（P0，几乎零成本）。
3. 抄复盘结构 + 补 akshare 数据字段 → digest 新增"A股收盘复盘"section（P1）。

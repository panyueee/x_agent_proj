# 人物逻辑提取 —— 多方法对照实验

> 目标：对不同类型的首席分析师，用**不同方法**提取其推理逻辑，并客观回答"哪种方法更适合哪类分析师"。
> 本目录是编排实验的方法学基座；各人物产物落 `output/personas/<person>/`，最终对比结论见 `comparison.md`。

## 0. 实验纪律（贯穿所有方法）

- **rag.db 只读**：一律 `sqlite3.connect("file:...?mode=ro", uri=True)`；库仍在被入库进程写，查询要容忍增长。
- **无 ANTHROPIC_API_KEY**：所有"LLM 充当分析师/裁判"的步骤都由 **sonnet 子 agent 手工执行**（读语料→写文件），不调 API。凡是 LLM 产物，文件头部标注 `LLM=sonnet subagent（手工执行，非 API）`。
- **不 git commit**；Python 一律 `.venv/bin/python`（3.14）。
- **防未来函数**：见 §2。提取（画像/规则/论点/判断）只能看截止日 `cutoff` 之前的语料；评估用其后的真实行情/政策/数据。
- **诚实**：语料不足或某人不适合某方法，如实写在其产物里；样本少的结论明确标注"管道验证，非统计结论"。

## 1. 五种方法定义

| 方法 | 名称 | 提取什么 | 怎么评 | 适合谁 |
|---|---|---|---|---|
| **A** | 数据接地机制 | if-then 因果机制规则 | 按日期 point-in-time 抓真实数据→现推→比对其真实结论 | 高频数据型（张瑜、明明） |
| **B** | 主线论点追踪 | 核心长期论点(thesis) + 时间线 | 沿时间线追一致性/转变点，看后续是否被兑现/证伪 | 长周期思辨型（李迅雷、戴康） |
| **C** | 可回测市场判断 | 显式行业/风格/大类资产判断 | 映射为持仓/方向→用真实市场收益打分（本项目独有能力） | 策略型（牟一凌、戴康） |
| **D** | 政策事件预测追踪 | 政策预测（定性+定量） | 对真实政策事件/宏观数据核对 | 财政/政策型（罗志恒） |
| **E**（基线） | 标签模仿 LLM 预测 | — | 画像→对测试窗口每篇文章去结论化预测→LLM-judge 命中率 | 已有系统，张瑜已做（严格命中 12.5%） |

### 方法 A（数据接地机制）
提取形如 `IF <可观测数据条件> THEN <该分析师会得出的判断>` 的机制规则集合；评估时按日期抓 point-in-time 真实数据代入规则现推，与其当期真实结论比对。张瑜的 A 由另一 agent 在 `output/personas/张瑜/grounded/` 落地（**本实验不动该目录**），此处借鉴其思路。

### 方法 B（主线论点追踪）
提取该分析师 2-5 条**跨时间的核心论点**（thesis），每条给：命题、首次提出时间、支撑逻辑、可证伪条件。评估 = 沿时间线检查论点的一致性（是否反复自我印证/悄悄转向）+ 论点是否被后续现实兑现或证伪。不依赖行情引擎，靠时间线内证 + 少量外部事实核对。B 对 A/C/D 的严格时间切分要求**豁免**（论点本身跨期），但时间线必须可排序。

### 方法 C（可回测市场判断）—— 重点做实
把"一次判断"严格定义为一个五元组：
```
{ date, target(行业/风格/资产), direction(看多/看空/中性), horizon, benchmark_type }
```
**打分分两层，职责分开**（回测引擎只支持多头、行和≤1、不能做空，故不能用引擎 P&L 直接给看空计分）：

1. **逐笔命中率（客观指标）**：直接用收盘价算 `sign(ret_target(h) − ret_benchmark(h))`（行业/风格判断）或 `sign(ret_target(h))`（纯大类资产判断，如债/黄金/纳指），与判断方向比对。看多看空对称处理，中性判断要求 |excess| 落在阈值带内算命中。horizon 固定 **20 和 60 个交易日**两档，不做"直到下一次相反判断"（管道demo从简）。
2. **引擎回测（叙事demo）**：只取**看多**判断，在其判断日给对应 ETF/行业篮子 weight=1.0，跑 `backtest.run_backtest`（成本/涨跌停沿用引擎默认），对比"跟随其看多判断"的 NAV vs 沪深300。这层给成本/涨跌停真实感，是管道验证不是统计结论。

target→可交易标的的映射（`scripts/pm_backtest_c.py` 内 `TARGET_MAP`）：优先行业/主题 ETF（`data/etf_history/`，有电力/旅游/农业/消费/黄金/有色/芯片/创业板/纳指等 53 只），大类资产用 `data/index_history/`（沪深300/创业板/恒生/纳指/…）与 `bond/crypto/fx`，无对应 ETF 的 GICS 行业用 `sw_sector_cache` 等权篮子兜底。

**逐笔命中率是 C 的核心可验证指标；引擎回测是加分的真实感演示。**

### 方法 D（政策事件预测追踪）
提取罗志恒对财政/政策的显式预测（定性方向 + 尽量带定量目标，如"赤字率提到 4%""专项债 X 万亿"）。评估：定量预测对 `data/macro_history/`（akshare 宏观 parquet）与其**截止日之后**的真实数据/后续文章核对；定性预测由 sonnet 子 agent 依据其后真实政策落地情况判 hit/partial/miss。**局限**：本项目无独立"政策事件时间线"数据库，D 的定性打分部分依赖子 agent 常识 + 事后语料，命中率偏乐观，如实标注。

## 2. 防未来函数协议（A/C/D 强制，B 豁免但时间线可排序）

- 语料日期来源（均已核实）：`wechat/column/research_web` → `extra_meta.date`（ISO `YYYY-MM-DD`）；`research` → 标题前缀 `YYYY-MM-DD`。无法解析日期的文章一律排除。
- **提取阶段只喂 `date < cutoff` 的语料**（子 agent prompt 里写死允许的日期上界）。
- **抗挑肥拣瘦（关键）**：C/D 的提取子 agent 必须抽出窗口内**全部**可定日期的显式判断/预测，不得只挑事后应验的；打分在提取之后由脚本**机械执行**，提取者无权决定哪条计入。
- **逐笔打分的行情/数据严格取判断日之后**：outcome 窗口 = `(call_date, call_date + h 交易日]`。
- 今天是 **2026-07-02**，前向窗口有限。各人 `cutoff` 选在数月前以保证 60 交易日的真实前向数据存在：牟一凌/罗志恒 `cutoff=2026-03-31`（前向到约 6 月底，数据齐全）；戴康语料截至 2026-05-05，逐笔判断只要求 `call_date + 60td ≤ 2026-07-02`（即判断日 ≤ ~2026-04-05）。

## 4. 正面对比设计

- **主对比：戴康 B vs C**（同一人、同一语料，两法各出结果）——直接比"论点追踪"与"可回测判断"在同一策略型分析师身上的表现。
- **参照对比：牟一凌 C vs 张瑜 E**——回答重构关键问题：可回测打分是否比已失败的标签模仿基线（12.5%）更有信息量。

评分三维度（见 `comparison.md`）：
- **可验证性**：能否客观、可复现地打分（有无未来函数、是否依赖 LLM 主观判断）。
- **还原度**：是否真的抓住了他的推理方式（而非套壳标签）。
- **预测价值**：产出对"预测他下一步"有没有实际用处。

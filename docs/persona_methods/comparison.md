> 方法学与协议见 `README.md`。本文汇总各方法在真实人物上的落地结果，按三维度打分，
> 给出"哪种方法适合哪类分析师"的结论表，以及对 `x_agent/persona/` 系统的重构建议。
> 所有 LLM 环节均由 **sonnet 子 agent 手工执行（非 API）**；方法C/A 的打分由脚本机械执行。

## 3. 正面对比

### 3.1 主对比：戴康 B vs C（同一人、同一语料）

| 维度 | 方法B（论点追踪） | 方法C（可回测判断） | 胜出 |
|---|---|---|---|
| 可验证性 | 弱：4条里仅黄金1条可独立强验证；"杠铃策略"等框架型命题几乎不可证伪 | 强：37条逐笔对真实行情打分，h60 严格48.6%/加权55.4%，全程机械可复现 | **C** |
| 还原度 | 高：完整重建其"逆全球化+债务周期+AI"三支柱框架与黄金信仰，并抓到"美国衰退"论点的静默转向 | 中：抽出37个方向判断，但丢掉了"为什么"——一个 target/direction 是其厚重逻辑的薄投影 | **B** |
| 预测价值 | 中：给出他的稳定先验（黄金必多、美股战略空）与行为模式（不复盘），利于预判"下一篇仍会说什么" | 中：给出方向胜率（h60 约五成，弱），可直接转化为持仓，但胜率本身不高 | 平 |

**结论**：对戴康这类**大类资产/风格策略师**，B 与 C 是**互补而非替代**——C 是客观计分板（他的判断到底赚不赚钱），B 是解释层（他为什么这么想、什么时候悄悄改口）。单看"哪种更好"要分维度：要客观分数用 C，要理解其人用 B。**方法B 最大的独家价值不是算胜率，而是揪出了单篇阅读发现不了的"叙事静默切换、不复盘"行为**——这是 C 完全看不到的。

### 3.2 参照对比：牟一凌 C vs 张瑜 E（回答重构关键问题）

- **E（张瑜标签模仿）**：严格命中 12.5%，失败模式是预测器对任何题目都套"结构性分化"标签，裁判还主观。
- **C（牟一凌可回测）**：严格命中 h20 43.3%/h60 40.0%，虽方向胜率也不高（策略行业判断本就难），但**每一分都来自真实行情、无裁判、防未来函数**，还能直接跑成 NAV（跟随其看多 -2.0% 超额，诚实反映其行业轮动约等于指数）。
- **两者非同一人，不能直接比数值高低**，但方法论结论明确：**只要分析师做的是可映射为持仓的市场判断，就应该用 C 式行情接地打分取代 E 式标签匹配**——C 把"预测对不对"变成"钱赚没赚"的客观问题，彻底绕开了 E 的两大病根（裁判主观 + 去结论化仍泄漏/仍可套标签）。

## 4. 结论表：哪种方法适合哪类分析师

| 分析师类型 | 代表 | 首选方法 | 理由 |
|---|---|---|---|
| 高频数据型 | 张瑜、明明 | **A**（+ C 于其资产判断部分） | 逐条数据点评→机制规则可现推、可算误差，验证最干净 |
| 策略/行业配置型 | 牟一凌、戴康 | **C**（客观分）+ **B**（解释层） | 判断可映射持仓→行情接地打分；B 补"为什么/行为漂移" |
| 长周期思辨型 | 李迅雷、戴康 | **B** | 少高频、多跨年论点；时间线追踪能抓一致性/转向/自我纠错 |
| 财政/政策型 | 罗志恒 | **D**（需补独立政策事件库） | 政策预测需对真实事件核对；当前受限于无事件库 |
| 任意（基线） | — | ~~E~~ | 标签模仿已被证明信息量低，应退役为对照基线 |

## 5. 对 persona 系统（`x_agent/persona/`）的重构建议

1. **按分析师类型路由方法，废弃 one-size-fits-all 的 E。** 在 `PERSON_SOURCES` 旁增设 `PERSON_METHOD` 路由表（本文结论表即初稿），`evaluate.py` 按类型分派到 A/B/C/D 评估器，E 仅保留为对照基线。

2. **把 C 的"判断五元组 + 行情接地打分"沉淀为一等评估器（最高优先级，本项目独有能力）。** 复用 `scripts/pm_backtest_c.py`（逐笔 excess 命中）+ `scripts/pm_engine_demo.py`（引擎 NAV）。它无 LLM 裁判、防未来函数最硬、看多看空对称，直接回答"判断赚不赚钱"。对所有策略型分析师优先用它替代 E。

3. **把 A 的"机制 if-then 规则 + point-in-time 数字点预测记分卡"制度化用于高频数据型。** 关键收益：无需裁判、算误差、且能暴露标签法看不见的结构（张瑜案例中"方向对、量级错、出口低估9pp"）。

4. **把 B 定位为"解释/行为层"而非胜率打分器。** 它的独家产出是：论点时间线一致性、**静默转向不复盘**（戴康案例）、**公开自我纠错**（李迅雷案例）。建议在画像里新增 `thesis_timeline` 与 `behavior_flags` 字段承载这些，不要试图给 B 强行算命中率。

5. **修复"标签化抽取制造伪预测"的画像 bug。** A 的 grounded 记分卡实测发现：画像把张瑜的 **PPI 预测值 3.5% 误挂到 CPI 名下**，凭空制造了一条错误预测。`profile.py` 的结构化抽取需加字段级出处（provenance）与回读校验，否则 E 的低命中率部分是抽取错误自造的。

6. **D 若要严肃化，须先建"政策事件时间线"数据来源**（财政方案/货币操作/两会目标的真实落地库），并在流程上强制：预测 vs 建议分离、剔除"下限式/nowcast"低信息量预测后再计命中率。当前 D 的命中率偏乐观，已在 `罗志恒/D_eval.md` 如实标注。

7. **预测题目生成弃用 E 的"去结论化"路线。** 该路线仍会泄漏主题、且预测器可套通用标签。改用 C 的思路：不问"你猜他会说什么"，而问"他的判断对应的持仓后来赚没赚"，用市场/数据当答案。

## 6. 诚实边界

- 各人样本量小（戴康仅51篇、C各30-37条判断、D仅6条预测），所有命中率为**管道验证，非统计结论**。
- C 的引擎回测受"首次判断前持币拖累""重叠等权""ETF 近似行业"等 demo 化简影响；逐笔 excess 命中率是更干净的核心指标。
- B 的兑现判定依赖子 agent 常识（截至其知识边界），2026 年中后段的地缘事件无法独立核实处已标注"未到期"。
- 方法 A 由另一 agent 在 `output/personas/张瑜/grounded/` 落地，本文仅只读引用，未改动。
